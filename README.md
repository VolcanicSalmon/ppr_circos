# trackplot

Interactive genome viewer for plant PPR small-RNA / phasiRNA target-cleavage results.
An openFrameworks (C++) app draws a circos overview of predicted slice sites with a
JBrowse-style two-pane genome-window ("GW") zoom, live phase-score/DC, and a per-window
**JB** button that opens a full JBrowse 2 at the same locus. The `bscore` scores it
plots come from `pval_dist.py`.

## Dependencies

The **only** thing you install by hand is openFrameworks ‚Äî the other two dependencies are
built/fetched automatically by `make` (see *Build & run*).

1. **openFrameworks 0.12.1** ‚Äî macOS release build from https://openframeworks.cc/download/.
   Not bundled here (large, separately licensed). Default expected at
   `/Applications/of_v0.12.1_osx_release`; override with `make OF_ROOT=/your/path`.
2. **`ofxDropdown`** (addon) ‚Äî fetched automatically: `make deps` clones it into
   `$OF_ROOT/addons/`. `ofxGui` (the other addon in `addons.make`) ships with OF.
3. **libBigWig** ‚Äî vendored in `libBigWig/`; built automatically (`-DNOCURL`, local `.bw`
   only). The `.a`/`.o` are git-ignored (arch-specific) and rebuilt on demand.
4. **Python 3** ‚Äî used two ways:
   - **Runtime (app):** the GW live phase/DC feature shells out to `gw_phase.py`, which
     needs **`pysam`**. The interpreter is hardcoded to `/opt/homebrew/bin/python3.14` in
     `src/ofApp.cpp` (macOS/Homebrew) ‚Äî edit it for other setups, or the feature no-ops.
   - **Data generation:** `pval_dist.py`, `make_bigwig.py`, `build_origins.py` need
     `pysam`, `pyBigWig`, `scipy`/`numpy`.
5. **JBrowse** ‚Äî a JBrowse 2 web build is **bundled** in `jbrowse/` (nothing to install).
   `serve.py` uses only the Python 3 standard library. Regenerating the local JBrowse
   tracks with `jbrowse/prep_tracks.sh` needs **`bedtools`**, **`bgzip`**, and **`tabix`**
   (htslib) on PATH.

## Build & run

```bash
make                              # builds libBigWig + clones ofxDropdown, then the app
                                  #   (override OF: make OF_ROOT=/path/to/of_v0.12.1_osx_release)
make RunRelease                   # launch (or open bin/trackplot.app)
```

`make` runs a `deps` step first (order-only): it builds `libBigWig/libBigWig.a` and clones
`ofxDropdown` into `$OF_ROOT/addons/` if missing ‚Äî so a fresh clone builds with a single
`make` once OF is installed. Run just the deps with `make deps`. `config.make` resolves
`PROJECT_PATH` from its own location, so the repo builds from any clone path; it does **not**
need to live inside `$OF_ROOT/apps/`.

## Data

Datasets under `bin/data/<name>/` (bigWigs, bscore TSV, origins, fasta, genome `.fai`,
BAM) are **not committed** (multi-GB). Only `bin/data/datasets.json` (the registry) is
tracked. Regenerate a dataset with the scoring pipeline (`pval_dist.py` + `make_bigwig.py`),
then add its entry to `datasets.json`. The app auto-discovers the transcript-region BED and
BAM under each `<name>/`.

## JBrowse (the JB button)

`jbrowse/serve.py` serves `jbrowse/` and reverse-proxies remote reference genomes (adds
CORS + a browser User-Agent, forwards `Range` so genomes stream and are never downloaded):

```bash
python3 jbrowse/serve.py 9000
```

Register a new genome for the JB button **without editing code** ‚Äî `add_jbrowse.py` edits
only `config.json`, `proxies.json`, and `datasets.json`:

```bash
python3 jbrowse/add_jbrowse.py --name <ds> --assembly <ASM> \
    --proxy-base https://their.host/ --fasta ref.fa --fai ref.fa.fai \
    [--gff models.gff3.gz --gff-tbi models.gff3.gz.tbi] [--aliases ...]
```

It handles `.fa`+`.fai` (IndexedFasta) or `.fa.gz`+`.fai`+`.gzi` (BgzipFasta), tabixed or
whole-file GFF, auto-discovers the dataset's local BED tracks, and prints the `bgzip`/`tabix`
prep commands. Restart `serve.py` afterward so the new proxy takes effect. Local JBrowse
tracks are generated by `jbrowse/prep_tracks.sh`.
